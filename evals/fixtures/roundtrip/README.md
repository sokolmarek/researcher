# Export round-trip fixtures (M5.4)

`cases.json` holds the hard-case bibliographic records that
`evals/run_roundtrip.py` and `core/tests/test_export.py` replay through every
export emitter. Each case is one CSL-JSON record (D4 canonical) plus a short
description; the runner round-trips it as `CSL -> emitter -> re-import -> CSL`
and checks that every field the format can carry survives byte for byte.

Everything here is offline: no network, no snapshots, no DOI resolution. The
`kessler2021cmae` record is the single sanctioned synthetic fake (D8); its DOI
must keep failing to resolve, and it is carried here as data only.

## The hard cases

| id | what it stresses |
| --- | --- |
| `diacritics-real` | Unicode diacritics in author names, a colon in the title (real DOI) |
| `braces-and-markup` | literal `{braces}` and XML-special `& < >` in title and abstract |
| `corporate-author` | an organization (literal) author with no comma |
| `no-doi` | a record with no DOI, so identity comes from its own fields |
| `kessler2021cmae` | the seeded `expect-unresolvable` fake (D8) |
| `datacite-dataset` | a real DataCite (Dryad) dataset DOI, CSL type `dataset` (D22) |
| `datacite-software` | a real DataCite (Zenodo) software DOI carrying a `version` (D22) |

The runner also builds a maximal probe in code that populates every comparable
field at once, so every structural loss below is exercised by a single record.

## Per-format loss table

CSL-JSON is canonical, so its round-trip is the identity and loses nothing.
BibTeX, RIS, and JATS `<ref-list>` are emitters; each carries most fields
losslessly and drops a documented few. A field a format cannot carry is listed
here, never dropped silently: the test suite fails on any undocumented loss, and
the runner fails if a documented loss is never actually observed.

| CSL field | CSL-JSON | RIS | JATS | BibTeX |
| --- | :---: | :---: | :---: | :---: |
| type | kept | kept | kept | dropped* |
| title | kept | kept | kept | kept** |
| author / editor | kept | kept | kept | kept |
| issued | kept | kept | kept | day dropped |
| container-title | kept | kept | kept | kept |
| publisher | kept | kept | kept | kept |
| volume / issue | kept | kept | kept | kept |
| page | kept | kept | kept | kept |
| number | kept | **dropped** | **dropped** | **dropped** |
| version | kept | **dropped** | kept | **dropped** |
| abstract | kept | kept | **dropped** | kept** |
| DOI | kept | kept | kept | kept |
| URL | kept | kept | kept | kept |
| ISSN | kept | kept | kept | **dropped** |
| ISBN | kept | dropped*** | kept | kept |
| language | kept | kept | **dropped** | kept |
| note | kept | kept | kept | kept |
| keyword | kept | kept | **dropped** | **dropped** |
| id | kept | kept | **dropped** | dropped* |

Notes:

- `*` BibTeX collapses non-native CSL types (`dataset`, `software`, `webpage`)
  to `document`, and replaces a DOI-shaped id with a generated citation key.
- `**` BibTeX consumes literal brace characters as capitalization-protection
  grouping, so a title or abstract that contains literal `{` `}` loses them;
  values without literal braces round-trip. This is the `braces-and-markup`
  fixture's per-case loss.
- `***` RIS has a single `SN` tag that holds the ISSN for serials; a record's
  ISBN is carried only when no ISSN competes for the tag (book-like items), so
  ISSN and ISBN cannot coexist in one RIS record.
- RIS represents a corporate author through the `AU` tag with a
  no-comma-multiword heuristic; a one-word organization and a mononym person are
  indistinguishable on import.
- BibTeX carries `issn`/`keywords` only under `custom.bibtex`, not as CSL
  variables (a schema constraint in `researcher_core.bib`), so they do not
  survive a CSL-origin round-trip.

The table above is generated by hand from `researcher_core.export.LOSS_TABLE`,
which is the single source of truth the eval and the tests assert against. Run
the gate with:

```
uv run --project core python evals/run_roundtrip.py
uv run --project core python evals/run_roundtrip.py --json
```
