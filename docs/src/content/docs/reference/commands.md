Slash commands are shortcuts into the most common workflows. Type them in Claude Code or Cowork.

| Command | What it does |
|---|---|
| `/new-manuscript` | Create a new manuscript project with the full folder structure. |
| `/draft-section <section>` | Draft a section (abstract, introduction, methods, results, discussion, conclusion). |
| `/review-paper` | Run the simulated multi-persona peer review. |
| `/submit-ready` | Pre-submission checklist: citations, formatting, word count, required sections. |
| `/revise <round>` | Handle a revision round (R1, R2, R3) with reviewer-comment parsing. |
| `/brainstorm` | Socratic research-design refinement. |
| `/find-journal` | Find best-fit journals for your paper. |
| `/find-conference` | Find relevant conferences with deadlines. |
| `/fact-check <claim>` | Verify a claim against the scientific literature. |
| `/sota <benchmark>` | Find state-of-the-art results for a benchmark. |
| `/design-experiment <question>` | Design an experiment for a research question. |

You never *have* to use a command. Plain language works too: "review my paper" fires the same skill as `/review-paper`. Commands just save keystrokes at 3 AM.
